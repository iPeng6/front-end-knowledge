集成百度地图

当然官方有文档http://developer.baidu.com/map/index.php?title=iossdk，这里做一下步骤记录

注册百度账号

注册成为百度开发者 http://developer.baidu.com/user/reg
iOS SDK首页 http://developer.baidu.com/map/index.php?title=iossdk
获取密钥－ 创建应用－安全码绑定Bundle Identifier

下载SDK http://developer.baidu.com/map/index.php?title=iossdk/sdkiosdev-download
运行demo，因为百度地图是绑定Bundle Id的所以运行需要复制demo工程的Bundle Id创建一个应用，复制AK粘贴到AppDelegate.mm的@”please enter your key”处

导入百度框架

将所需的BaiduMapAPI_**.framework拷贝到工程所在文件夹下

TARGETS->Build Phases-> Link Binary With Libaries => “+” => “Add Other”=> BaiduMapAPI_**.framework

将 AppDelegate.m 改名 => AppDelegate.mm

导入系统库

Project -> Active Target ->Build Phases ->Link Binary With Libraries => +

BaiduMapAPI_Map.framework||Resources -> mapapi.bundle -> 添加到工程 ->“Copy items if needed”

ATS配置使可Http网络访问

Project -> Edit Active Target -> Build Settings-> Linking -> Other Linker Flags中添加-ObjC

下面开始走坑了，参考地址集成原生UI组件

1. 坑一

经过摸索我发现这个ui的命名是有讲究的，我一开始直接胡乱命名了一个测试视图管理类，我本以为只要继承管理类 遵循协议 实现方法 就都可以的，而且我觉得百度已经有一个BMKMapView我直接调用就可以，唯一要做的就是写一个管理模块暴露一些属性和方法即可，而事实测试发现这里有个大坑

原来这里的文件组织命名是有讲究的否则会遇到各种问题，在翻看了原生mapview封装的js
(路径node_modules/react-native/Libraries/Components/MapView/MapView.js)

发现他导入的组件是RCTMap，不是RCTMapManager，而我偷懒只创建了一个管理类，如果我直接导入BMKMapView它是怎么知道我对应的管理类就是我创建的文件BDViewMgr的呢？

经过多次测试发现，管理类一定要是：视图名称+Manager，所以我还得必须创建一个视图，然后js里导入这个视图，而ReactNative会自动找到它对应的Manager类

3. 坑三

在这之前我也尝试过将自己的类名命名成RCTBDMapManager，后来发现这里也有一个坑

当我React.NativeModules.RCTBDMapManager这样想导入模块的时候始终为undefine，我找了好久不知道原因，后来无意中我把RCT去掉后 可以了

我们一直以为NativeModules后面直接接的模块原名，原来系统还会将RCT做一个过滤，RCT可能表示系统的，系统里查不到，所以以后命名自定义的别以RCT开头
