老规矩第一篇建站日志

这里做一下步骤记录，本站通过hexo+github搭建

Mac安装Hexo博客

官方教程

https://hexo.io/zh-cn/docs/index.html

安装Node.js

https://nodejs.org/en/

查看版本 node －v

安装Git,安装了Xcode即自动安装了Git

查看版本 git version

安装hexo

npm install -g hexo-cli

查看版本 hexo -v

建站

hexo init

cd

npm install

安装服务器hexo-server

npm install hexo-server –save

启动服务器

hexo server

本地测试

http://0.0.0.0:4000/hexoblog/

生成静态页面

hexo generate

配置GitHub

到GitHub上new repository（例：hexoblog）,复制地址

配置_config.yml

安装 hexo-deployer-git

npm install hexo-deployer-git –save

部署

hexo deploy

生成后部署

hexo generate –deploy //简写 hexo d -g

外网访问

http://yourname.github.io/hexoblog/

新建文章

hexo new [layout]

域名绑定

在仓库根目录瞎新建CNAME文件，写上你要绑定的域名如

qq.cn

在域名提供商后台添加A记录 指向192.30.252.153或者192.30.252.154

注意不要重复设置两个

  1. 不要让相同记录指向不同的ip

  2. 也不要多个记录指向同一个博客 搜索引擎会认为是不同的站点 会分散权重

解决方法 通过301重定向 可以保证权重

如qq.cn => 192.30.252.153

再新建一个CNAME记录

www.qq.cn => qq.github.io 因为我们知道qq.github.io这个已经是被301永久重定向qq.cn的

主题修改

主题使用的spfk
配置多说评论 将多说二级域名的第一部分 作为short_name填入配置文件
将clone下来的主题里的多说.css文件 加入到多说后台自定义配置中
